2. **Add truncate_error_log() function:**
Truncates error.log to the last 1000 lines when it exceeds 5000 lines. Uses `tail -n 1000` + `mv` pattern (same as tunnel watchdog log). Call pattern: called at the top of bin/cc-tmux (Plan 02 will wire this). Implementation:
```bash
truncate_error_log() {
    local log_file="$HOME/.cc-tmux/error.log"
    if [[ -f "$log_file" ]]; then
        local line_count
        line_count=$(wc -l < "$log_file" 2>/dev/null) || return 0
        if [[ "$line_count" -gt 5000 ]]; then
            tail -n 1000 "$log_file" > "${log_file}.tmp"
            mv "${log_file}.tmp" "$log_file"
        fi
    fi
}
```

3. **Add remove_bashrc_block() function** (counterpart to add_bashrc_block):
Uses sed to delete between sentinel markers. Detection with `grep -qF` (fixed string). Deletion with `sed -i "/CC-TMUX:${name}:START/,/CC-TMUX:${name}:END/d"`. Log success or "not found (already removed?)".

Note: [FAIL] is uppercase to stand out. [pass] is lowercase (same as existing step_verify pattern).

**run_doctor() function** -- the orchestrator:

```bash
run_doctor() {
    local pass=0 fail=0

    echo ""
    echo "${BOLD}CC-TMUX Health Check${RESET}"
    echo "--------------------"
    echo ""

    check_wsl              && ((pass++)) || ((fail++))
    check_tmux             && ((pass++)) || ((fail++))
    check_ssh_service      && ((pass++)) || ((fail++))
    check_sshd_config      && ((pass++)) || ((fail++))
    check_ssh_keys         && ((pass++)) || ((fail++))
    check_fail2ban         && ((pass++)) || ((fail++))
    check_ngrok            && ((pass++)) || ((fail++))
    check_tunnel_files     && ((pass++)) || ((fail++))
    check_config           && ((pass++)) || ((fail++))
    check_projects         && ((pass++)) || ((fail++))
    check_directory_structure && ((pass++)) || ((fail++))
    check_tmux_conf        && ((pass++)) || ((fail++))
    check_cli              && ((pass++)) || ((fail++))

    echo ""
    echo "  Results: ${GREEN}$pass passed${RESET}, ${RED}$fail failed${RESET}"
    echo ""

    [[ $fail -eq 0 ]]
}
```

The final `[[ $fail -eq 0 ]]` ensures exit code 0 when all pass, 1 when any fail.

**Important:** Doctor diagnoses only. It does NOT run any apt install, service start, or config modification. Each failure includes a "Fix:" hint that tells the user what command to run. This is a locked user decision.
