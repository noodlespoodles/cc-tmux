**Section 4: Phone Setup** (heading: `## Phone Setup`)
- During installation, an SSH private key was displayed. Explain how to get it into Termius:
  1. The key was shown during install. If you need it again: `cat ~/.cc-tmux/keys/cc-tmux_ed25519`
  2. Copy the key text (everything from `-----BEGIN` to `-----END`).
  3. In Termius: Keychain -> + -> Key -> paste key text, save.
  4. Create a new Host in Termius:
     - Hostname and Port: shown each time you start the workspace (or scan the QR code)
     - Username: your WSL username (the one from Step 1)
     - Key: select the key you just imported
  5. Note: the hostname and port change each time you reboot. The QR code at startup makes this easy.

**Section 5: Daily Usage** (heading: `## Daily Usage`)
- **Starting up**: Double-click "Claude Workspace" on desktop. Or in WSL: `cc-tmux start`. It starts SSH, tunnel, and drops you into your tabbed workspace with a QR code for your phone.
- **On your PC**: Click tabs at bottom, click + for new tab, type `claude` to start Claude Code, just close the window when done -- everything keeps running.
- **On your phone**: Open Termius, update the port if it changed (or scan the QR code on your PC screen), tap to connect, you land in your workspace automatically. Ctrl+B, Shift+M for mobile mode (bigger tabs). Close Termius when done -- everything keeps running.
- **When you leave**: Just walk away. Open Termius whenever you want to check in.
- **When you get back**: Open the desktop shortcut or WSL and run `cc-tmux start`. Everything is exactly where you left it.

**Section 7: Troubleshooting** (heading: `## Troubleshooting`)
- First line: "**First step for any issue:** Run `cc-tmux doctor`. It checks every component and tells you exactly what's wrong."
- Then cover these 8 scenarios with heading + 2-3 line solution with exact commands:
  1. **Can't connect from Termius** -- PC on/awake? Port changed? (cc-tmux tunnel to check). VPN interference?
  2. **SSH won't start** -- `sudo service ssh restart && sudo service ssh status`
  3. **ngrok isn't running** -- `cc-tmux start` to restart everything, or check `cc-tmux tunnel`
  4. **tmux session disappeared** -- PC probably restarted. Double-click desktop shortcut or `cc-tmux start`.
  5. **PowerShell won't start in a tab** -- Check: `which powershell.exe`. Should return a path under /mnt/c/.
  6. **I closed the window by accident** -- No problem. `cc-tmux start` reattaches. Everything is still running.
  7. **My PC went to sleep** -- WSL shuts down on sleep. Change power settings: Settings -> System -> Power -> Sleep -> Never (when plugged in). Then `cc-tmux start`.
  8. **"Permission denied" on SSH** -- SSH key not imported into Termius. See Phone Setup section. Run `cat ~/.cc-tmux/keys/cc-tmux_ed25519` to display the key again.

**Section 8: Uninstalling** (heading: `## Uninstalling`)
- One-liner: `cc-tmux uninstall`
- Brief explanation: removes configs, bashrc hooks, desktop shortcut, ~/.cc-tmux directory. Does NOT remove system packages (tmux, ngrok, etc.) -- shows the apt remove command if they want to.
