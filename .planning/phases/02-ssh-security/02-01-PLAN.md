**Function 1: generate_ssh_keys()**
- Set `local key_dir="$CC_TMUX_DIR/keys"` and `local key_path="$key_dir/cc-tmux_ed25519"`
- `mkdir -p "$key_dir"` and `chmod 700 "$key_dir"`
- If `[[ -f "$key_path" ]]`, call `log_ok "SSH key pair already exists"` and return 0
- Generate: `ssh-keygen -t ed25519 -f "$key_path" -N "" -C "cc-tmux-$(date +%Y%m%d)"`
- Set permissions: `chmod 600 "$key_path"` and `chmod 644 "${key_path}.pub"`
- Save to config: `write_config CC_TMUX_SSH_KEY "$key_path"`
- `log_ok "Ed25519 key pair generated at $key_path"`

**Function 2: install_public_key()**
- Set `local pub_key_path="$CC_TMUX_DIR/keys/cc-tmux_ed25519.pub"`
- Set `local auth_keys="$HOME/.ssh/authorized_keys"`
- `mkdir -p "$HOME/.ssh"` and `chmod 700 "$HOME/.ssh"`
- Read: `local pub_key; pub_key=$(cat "$pub_key_path")`
- If `grep -qF "$pub_key" "$auth_keys" 2>/dev/null`, call `log_ok "Public key already in authorized_keys"` and return
- Else: `echo "$pub_key" >> "$auth_keys"` and `chmod 600 "$auth_keys"`
- `log_ok "Public key added to authorized_keys"`

**Function 3: display_key_instructions()**
- Takes one argument: `local key_path="$1"`
- Print a boxed display using echo statements (NO color codes in the key output itself):
  - Header: "========================================"
  - Title: "  Your SSH Private Key"
  - "========================================"
  - Blank line
  - WHY section: "  WHY: Your SSH connection now requires" / "  a key instead of a password. This is" / "  much more secure -- like a lock that" / "  only your specific key can open."
  - Blank line
  - "  Copy everything between the lines below:"
  - Blank line
  - "  ---- BEGIN KEY (copy from here) ----"
  - `cat "$key_path"` (the actual private key content)
  - "  ---- END KEY (copy to here) ----"
  - Blank line
  - HOW TO IMPORT section with numbered steps 1-8 for Termius Android:
    1. Open Termius on your phone
    2. Tap Settings (gear icon)
    3. Tap Keychain
    4. Tap + (plus) to add a new key
    5. Tap 'Paste from clipboard'
    6. Paste the key you copied above
    7. Give it a name like 'cc-tmux'
    8. Tap Save
  - Blank line
  - "  Then when setting up your SSH connection"
  - "  in Termius, select this key instead of"
  - "  using a password."
  - Blank line
  - "========================================"

**Function 4: write_hardened_ssh_config()**
- Get current user: `local current_user; current_user=$(whoami)`
- Set config path: `local conf="/etc/ssh/sshd_config.d/00-cc-tmux.conf"`
- Remove the old Phase 1 drop-in if it exists (different filename): `sudo rm -f /etc/ssh/sshd_config.d/cc-tmux.conf`
- Write the hardened config using `sudo tee "$conf" > /dev/null` with a heredoc. The EXACT content must be:
```
# Managed by cc-tmux installer -- Phase 2 hardened config
# Do not edit manually; re-run installer to regenerate

ListenAddress 0.0.0.0

# Authentication -- key-only for remote connections
PubkeyAuthentication yes
PasswordAuthentication no
PermitRootLogin no
MaxAuthTries 3
LoginGraceTime 30
PermitEmptyPasswords no

# Access control
AllowUsers {current_user}

# Security hardening
X11Forwarding no
ClientAliveInterval 120
ClientAliveCountMax 3

# Localhost safety net -- password auth on loopback
Match Address 127.0.0.1,::1
    PasswordAuthentication yes
```
Where `{current_user}` is replaced by the `$current_user` variable.

CRITICAL: Do NOT include `Protocol 2` -- it was removed from OpenSSH 7.6 and will cause sshd to fail.

CRITICAL: The `Match Address` block MUST be the LAST thing in the file -- no global directives after it.

- After writing, validate with `sudo sshd -t`:
  - If valid: `log_ok "SSH config syntax valid"`
  - If invalid: `log_error "SSH config syntax invalid -- rolling back"`, restore a minimal safe config (ListenAddress 0.0.0.0, PasswordAuthentication yes, PubkeyAuthentication yes) to the SAME file path (`00-cc-tmux.conf`), and return 1
- Restart SSH: `sudo service ssh restart`
- `log_ok "SSH hardened: key-only auth (password on localhost only)"`

**Function 5: configure_fail2ban()**
- Set jail path: `local jail_conf="/etc/fail2ban/jail.d/cc-tmux.conf"`
- Auto-detect backend:
  - If `/var/log/auth.log` exists: `local backend="auto"`
  - Elif `systemctl is-active systemd-journald &>/dev/null` succeeds: `local backend="systemd"`
  - Else: `local backend="auto"` (fallback)
- Write jail config using `sudo tee "$jail_conf" > /dev/null`:
```
# Managed by cc-tmux installer
[sshd]
enabled = true
port = ssh
filter = sshd
backend = {backend}
maxretry = 5
bantime = 600
findtime = 600
```
Where `{backend}` is replaced by the detected `$backend` variable.
- Enable and start fail2ban: `sudo service fail2ban restart`
- Verify: `if sudo fail2ban-client status sshd &>/dev/null; then log_ok "fail2ban sshd jail active"; else log_warn "fail2ban sshd jail not responding -- may need manual check"; fi`

**Function 6: step_harden_ssh()**
This is the orchestrator function that Plan 02 will call from install.sh. It runs the full sequence:

1. `generate_ssh_keys`
2. `install_public_key`
3. `display_key_instructions "$CC_TMUX_DIR/keys/cc-tmux_ed25519"`
4. Interactive confirmation (check `$NONINTERACTIVE`):
   - If `NONINTERACTIVE` is NOT true:
     - Print warning: `echo ""; echo "  ${YELLOW}[!]${RESET} Password authentication will be DISABLED for remote SSH."; echo "      Make sure you have copied the key above to your phone."; echo ""`
     - Prompt: `read -rp "  Continue? (Y/n): " confirm`
     - If user enters "n" or "N": `log_warn "Skipped SSH hardening. Password auth remains enabled."; return 0`
   - If `NONINTERACTIVE` is true: proceed immediately (no prompt)
5. `write_hardened_ssh_config`
6. `configure_fail2ban`
7. `log_ok "SSH security hardening complete"`
