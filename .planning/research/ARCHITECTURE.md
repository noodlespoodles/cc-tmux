# Architecture Research: CC x TMUX V2

**Domain:** Bash CLI toolkit architecture
**Researched:** 2026-03-20
**Confidence:** HIGH

## Component Architecture

```
cc-tmux/
├── install.sh              # Entry point — idempotent installer
├── uninstall.sh            # Clean removal
├── bin/
│   └── cc-tmux             # Main CLI entry point (subcommand router)
├── lib/
│   ├── common.sh           # Shared utilities (logging, colors, validation)
│   ├── config.sh           # Config read/write (config.env)
│   ├── detect.sh           # Auto-detection (Windows user, WSL distro, paths)
│   ├── ssh.sh              # SSH setup, key generation, hardening
│   ├── tmux.sh             # tmux config generation, session management
│   └── tunnel/
│       ├── provider.sh     # Tunnel provider interface (start/stop/status/url)
│       ├── ngrok.sh        # ngrok implementation
│       └── tailscale.sh    # Tailscale implementation (future)
├── templates/
│   ├── tmux.conf.tpl       # tmux config template (variable substitution)
│   ├── sshd_workspace.conf # SSH hardening config
│   └── bashrc-hook.sh      # Auto-attach snippet for .bashrc
├── config.env              # Generated by installer (user-specific values)
└── README.md
```

## Component Boundaries

| Component | Responsibility | Depends On |
|-----------|---------------|------------|
| `install.sh` | Orchestrates setup, installs deps, calls lib modules | detect.sh, config.sh |
| `bin/cc-tmux` | CLI router — `cc-tmux start`, `cc-tmux project add`, etc. | All lib/ modules |
| `lib/common.sh` | Logging, colors, error handling, path utilities | Nothing |
| `lib/config.sh` | Read/write config.env, validate settings | common.sh |
| `lib/detect.sh` | Auto-detect Windows username, WSL distro, paths | common.sh |
| `lib/ssh.sh` | SSH key gen, sshd config, fail2ban setup | common.sh, config.sh |
| `lib/tmux.sh` | Generate tmux.conf, manage sessions, project tabs | common.sh, config.sh |
| `lib/tunnel/provider.sh` | Abstract tunnel interface | common.sh, config.sh |
| `lib/tunnel/ngrok.sh` | ngrok-specific start/stop/status/url | provider.sh |

## Data Flow

```
User runs install.sh
  → detect.sh discovers Windows username, paths, WSL distro
  → Prompts user for ngrok token, project folders
  → config.sh writes config.env with all discovered + user values
  → ssh.sh generates Ed25519 keys, writes sshd config
  → tmux.sh generates tmux.conf from template + config values
  → Writes .bashrc auto-attach hook
  → Creates desktop shortcut via PowerShell

User runs cc-tmux start
  → config.sh loads config.env
  → ssh.sh ensures sshd is running
  → tmux.sh creates/attaches session with project windows
  → tunnel/provider.sh starts configured tunnel
  → Displays connection info

User runs cc-tmux project add "name" "/path"
  → config.sh reads/updates project list in config.env
  → tmux.sh creates new window in running session (if active)
```

## Key Patterns

**Sourced modules:** All lib/ files are sourced (`. lib/common.sh`), not executed as subprocesses. This shares state and avoids subprocess overhead.

**Config as environment:** `config.env` is a sourceable bash file (`KEY=value` format). No YAML/JSON parser needed. Simple, robust, zero dependencies.

**Template substitution:** tmux.conf uses `envsubst` or simple `sed` replacement from config values. Templates are readable; generated output is not meant to be hand-edited.

**Idempotent operations:** Every function checks current state before acting. `install.sh` can be run repeatedly without damage. Each step is a function that can be called independently.

**Tunnel provider interface:** Each tunnel provider implements 4 functions: `tunnel_start()`, `tunnel_stop()`, `tunnel_status()`, `tunnel_url()`. The provider.sh sources the configured provider module.

## Build Order

1. **lib/common.sh** — foundation for everything else
2. **lib/detect.sh + lib/config.sh** — auto-detection and config storage
3. **install.sh** — installer using detect + config
4. **lib/ssh.sh** — SSH hardening (must work before tunnel)
5. **lib/tunnel/** — tunnel abstraction + ngrok provider
6. **lib/tmux.sh** — workspace management
7. **bin/cc-tmux** — CLI wrapper tying it all together
8. **Polish** — desktop shortcut, health check, uninstall, README
