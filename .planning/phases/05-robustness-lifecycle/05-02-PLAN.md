**lib/update.sh:**

Header comment following established pattern (see lib/config.sh).

Implement these functions:

1. `check_for_updates(repo_dir)` -- Compare local vs remote HEAD:
   - Validate `$repo_dir/.git` exists, log_error if not
   - `local_head=$(git -C "$repo_dir" rev-parse HEAD 2>/dev/null)`
   - `remote_head=$(timeout 10 git -C "$repo_dir" ls-remote origin HEAD 2>/dev/null | cut -f1)`
   - If remote_head is empty: `log_warn "Could not reach remote -- check your internet connection"` and return 1
   - If local == remote: `log_ok "Already up to date (${local_head:0:8})"` and return 0
   - If different: show local/remote short hashes, return 2 (signal: update available)

2. `handle_dirty_repo(repo_dir)` -- Check for uncommitted changes:
   - If `git -C "$repo_dir" status --porcelain` is non-empty: warn and offer stash (option 1) or abort (option 2)
   - On stash: `git -C "$repo_dir" stash` and return 0
   - On abort: return 1
   - If clean: return 0 silently

3. `run_update()` -- Main update orchestrator:
   - Get repo path: `local repo_dir; repo_dir=$(get_config "CC_TMUX_REPO" 2>/dev/null)`
   - If empty or dir doesn't exist: `log_error "Repository path not found in config."` then `log_hint "If you moved the repo, update CC_TMUX_REPO in ~/.cc-tmux/config.env"` and return 1
   - Call `check_for_updates "$repo_dir"` -- capture return code
   - If return 0 (up to date): return 0
   - If return 1 (network error): return 1
   - If return 2 (update available): proceed
   - Call `handle_dirty_repo "$repo_dir"` -- abort if returns 1
   - Backup config: `cp "$CC_TMUX_DIR/config.env" "$CC_TMUX_DIR/config.env.bak"` and `cp "$CC_TMUX_DIR/projects.conf" "$CC_TMUX_DIR/projects.conf.bak" 2>/dev/null || true`
   - `log_ok "Config backed up (config.env.bak, projects.conf.bak)"`
   - Pull: `git -C "$repo_dir" pull --ff-only` (use --ff-only to avoid merge conflicts). If pull fails, log_error and return 1.
   - Re-deploy from freshly pulled repo: Source `"$repo_dir/lib/common.sh"` and `"$repo_dir/lib/setup.sh"` from the REPO (not from ~/.cc-tmux/), then call `step_deploy`. This is critical because the running script is ~/.cc-tmux/bin/cc-tmux but step_deploy needs to find files relative to the repo.
   - Restore configs: `cp "$CC_TMUX_DIR/config.env.bak" "$CC_TMUX_DIR/config.env"` and same for projects.conf
   - `log_ok "Update complete! Restart your workspace to use the new version."`

**lib/uninstall.sh:**

Header comment following established pattern.

Implement:

1. `run_uninstall()` -- Ordered teardown with confirmation:
   - Parse `--yes` / `-y` flag from `$1`
   - Show what will be removed (exact list from CONTEXT.md decisions):
     ```
     The following will be removed:
       - ~/.cc-tmux/ directory (config, libs, templates)
       - bashrc hooks (auto-attach, path)
       - /etc/sudoers.d/cc-tmux
       - /etc/ssh/sshd_config.d/00-cc-tmux.conf

   - **Phase 1 -- Stop running services** (needs ~/.cc-tmux files still):
     - Get session name: `session_name=$(get_config "SESSION_NAME" 2>/dev/null) || session_name="work"`

     - Kill tmux session: `tmux kill-session -t "$session_name" 2>/dev/null || true`

doctor)
    if [[ ! -f "$CC_TMUX_DIR/lib/doctor.sh" ]]; then
        log_error "doctor module not found -- try reinstalling"

    fi
    source "$CC_TMUX_DIR/lib/doctor.sh"

    ;;

update)
    if [[ ! -f "$CC_TMUX_DIR/lib/update.sh" ]]; then
        log_error "update module not found -- try reinstalling"
        exit 1
    fi
    source "$CC_TMUX_DIR/lib/update.sh"
    run_update
    ;;

uninstall)
    if [[ ! -f "$CC_TMUX_DIR/lib/uninstall.sh" ]]; then
        log_error "uninstall module not found -- try reinstalling"
        exit 1

    source "$CC_TMUX_DIR/lib/uninstall.sh"
    shift
    run_uninstall "${1:-}"
    ;;

version)
    local_ver=$(get_config "CC_TMUX_VERSION" 2>/dev/null) || local_ver="unknown"
    echo "cc-tmux $local_ver"
    ;;
```
