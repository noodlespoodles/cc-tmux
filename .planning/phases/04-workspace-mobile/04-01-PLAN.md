  <action>
Create `lib/workspace.sh` with two functions:

**workspace_init():**
- Read session name from config: `get_config "SESSION_NAME"` with fallback to "work"
- If `tmux has-session -t "$session_name"` succeeds, return 0 (idempotent)
- Read `$CC_TMUX_DIR/projects.conf` (same as PROJECTS_FILE)
- If file missing or empty: create single window named "default" in WIN_HOME Documents folder (get WIN_HOME from config, fallback to `/mnt/c/Users/$USER/Documents`), launch powershell.exe via send-keys, return 0
- First project: `tmux new-session -d -s "$session_name" -n "$name" -c "$path"` then `tmux send-keys -t "$session_name:$name" "powershell.exe" Enter`
- Remaining projects: `tmux new-window -t "$session_name" -n "$name" -c "$path"` then send-keys powershell.exe Enter
- After loop: `tmux select-window -t "$session_name:1"`
- Use `while IFS='|' read -r name path` loop to parse projects.conf (same format as config.sh uses)
- Skip empty lines and lines where name is empty

**workspace_attach():**
- Read session name from config with fallback to "work"
- Call workspace_init (ensures session exists)
- `exec tmux attach -t "$session_name"`

File header follows established pattern: `#!/usr/bin/env bash` comment block describing purpose, "Assumes lib/common.sh and lib/config.sh have already been sourced"

Create `templates/mobile-check.sh`:
- `#!/usr/bin/env bash`
- Comment: "Mobile auto-detection for tmux -- called by set-hook client-attached in tmux.conf"
- `WIDTH=$(tmux display -p '#{client_width}')`
- If WIDTH < 80: apply mobile styling via tmux set commands:
  - `tmux set status-left ""`
  - `set status-right "#[bg=#89b4fa,fg=#1e1e2e,bold] + "`
  - `set status-right-length 5`
  - `setw window-status-format "#[bg=#45475a,fg=#cdd6f4]  #I: #W  "` (extra padding for tap targets)
  - `setw window-status-current-format "#[bg=#89b4fa,fg=#1e1e2e,bold]  #I: #W  "` (extra padding)
  - `setw window-status-separator " "` (space between tabs for tap targets)
  - `display-message "Mobile mode"`
- All tmux set commands chained with `\;` on one tmux invocation
- If WIDTH >= 80: do nothing (desktop is the default from tmux.conf)
- Comment explaining: "If width >= 80, desktop layout is already active from tmux.conf"
  </action>

Sections in order:

2. **Quality of life:**
   - `history-limit 50000`
   - `display-time 3000`
   - `status-interval 5`
   - `default-terminal "tmux-256color"`
   - `focus-events on`
   - `escape-time 0`

3. **Window numbering:**
   - `base-index 1`
   - `pane-base-index 1`
   - `renumber-windows on`

4. **Status bar (Catppuccin-inspired):**
   - `status-position bottom`
   - `status-style "bg=#1e1e2e,fg=#cdd6f4"`
   - `status-left "#[bg=#89b4fa,fg=#1e1e2e,bold]  #S "`
   - `status-left-length 30`
   - `status-right "#[bg=#89b4fa,fg=#1e1e2e,bold] + #[default] #[fg=#a6adc8]%H:%M  %d-%b "`
   - `status-right-length 30`

5. **Tab styling:**
   - `window-status-format " #I:#W "`
   - `window-status-current-format "#[bg=#89b4fa,fg=#1e1e2e,bold] #I:#W "`
   - `window-status-separator ""`

6. **Keybindings:**
   - `bind | split-window -h -c "#{pane_current_path}"`
   - `bind - split-window -v -c "#{pane_current_path}"`
   - `bind R source-file ~/.tmux.conf \; display-message "Config reloaded"`
   - `bind c new-window -c "#{pane_current_path}"`
   - `bind -n M-Left previous-window`
   - `bind -n M-Right next-window`

7. **Clickable + button:** Uses `__USERNAME__` placeholder:
   `bind -n MouseDown1StatusRight new-window -c "/mnt/c/Users/__USERNAME__/Documents" \; send-keys "powershell.exe" Enter`

8. **Mobile auto-detect on attach:**
   `set-hook -g client-attached 'run-shell "~/.cc-tmux/templates/mobile-check.sh"'`

9. **Mobile mode toggle (Ctrl+B, Shift+M):**
   Same bind M command as V1 -- sets empty status-left, minimal status-right with just +, wider tab formats with extra spacing, window-status-separator " ", display-message "Mobile mode"

10. **Desktop mode toggle (Ctrl+B, Shift+N):**
    `bind N source-file ~/.tmux.conf \; display-message "Desktop mode"`

The `__USERNAME__` placeholder will be replaced by sed during install (Plan 02 handles deployment). Use exact same hex colors as V1: bg=#1e1e2e, fg=#cdd6f4, accent=#89b4fa, inactive tab bg=#45475a, muted text=#a6adc8.
