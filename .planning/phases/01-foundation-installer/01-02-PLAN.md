Create `lib/deps.sh` with these functions:

1. `step_install_deps()` -- Installs apt packages. Calls `log_step` with current step number. Runs `sudo apt update -qq` once at the start. Then calls `install_package` (from common.sh) for each: tmux, openssh-server, jq, fail2ban. Each package gets its own install_package call so status is shown per-package.

2. `install_ngrok()` -- Installs ngrok via apt repository (NOT snap). First checks `command -v ngrok &>/dev/null`. If already installed, logs ok with version (`ngrok version 2>/dev/null || echo 'unknown'`). If not installed:
   - Downloads GPG key: `curl -sSL https://ngrok-agent.s3.amazonaws.com/ngrok.asc | sudo tee /etc/apt/trusted.gpg.d/ngrok.asc >/dev/null`
   - Adds apt repo: `echo "deb https://ngrok-agent.s3.amazonaws.com bookworm main" | sudo tee /etc/apt/sources.list.d/ngrok.list >/dev/null`
   - Runs `sudo apt update -qq && sudo apt install -y ngrok`
   - Error handling with log_error and log_hint on each step

3. `step_install_ngrok()` -- Calls `log_step` with step number, then calls `install_ngrok`.

4. `setup_ngrok_token()` -- Checks if ngrok token is configured by testing `ngrok config check &>/dev/null`. If configured, logs ok and returns. If not configured:
   - Prints instructions: signup URL `https://ngrok.com`, token URL `https://dashboard.ngrok.com/get-started/your-authtoken`
   - If `$NONINTERACTIVE` is true, logs warning "Skipped in non-interactive mode. Run later: ngrok config add-authtoken YOUR_TOKEN" and returns
   - Otherwise: `read -rp "  Paste your ngrok auth token (or press Enter to skip): " token`
   - If token provided: `ngrok config add-authtoken "$token"` with error handling
   - If empty: logs warning with how to add later

Create `lib/setup.sh` with these functions:

1. `setup_ssh_config()` -- Writes SSH config drop-in to `/etc/ssh/sshd_config.d/cc-tmux.conf` (not workspace.conf -- namespaced to cc-tmux). Content:
   ```
   # Managed by cc-tmux installer
   ListenAddress 0.0.0.0
   PasswordAuthentication yes
   PubkeyAuthentication yes
   ```
   Uses `sudo tee` to write. Runs `sudo ssh-keygen -A 2>/dev/null` to generate host keys if missing. Restarts SSH with `sudo service ssh restart`. Note: Password auth is kept for Phase 1 convenience -- Phase 2 hardens to key-only.

2. `setup_sudoers()` -- Writes sudoers file to `/etc/sudoers.d/cc-tmux`. Content: `$USER ALL=(ALL) NOPASSWD: /usr/sbin/service ssh start, /usr/sbin/service ssh restart, /usr/sbin/service ssh status` (restricted commands, not wildcard). Uses `sudo tee` to write to a temp file first, validates with `sudo visudo -c -f /tmp/cc-tmux-sudoers`, then moves to `/etc/sudoers.d/cc-tmux`. Sets permissions to 0440. If validation fails, removes temp file and logs error.

3. `setup_bashrc_hook()` -- Uses `add_bashrc_block` (from common.sh) with block name "auto-attach" and content that sources the bashrc-hook template:
   ```
   if [ -n "$SSH_CONNECTION" ] && [ -z "$TMUX" ]; then
       source "$HOME/.cc-tmux/templates/bashrc-hook.sh"
   fi
   ```

4. `step_deploy()` -- Calls `log_step` with step number. Copies lib/ and templates/ from repo to `$CC_TMUX_DIR/` using `deploy_file`. Creates `$CC_TMUX_DIR/lib/` and `$CC_TMUX_DIR/templates/` directories. Copies each lib/*.sh and templates/* file. Sets lib/*.sh files to 644 (they are sourced, not executed directly).

5. `step_verify()` -- Calls `log_step` with step number. Verifies:
   - `command -v tmux` (tmux installed)
   - `command -v ngrok` (ngrok installed)
   - `command -v jq` (jq installed)
   - `dpkg -l fail2ban | grep -q "^ii"` (fail2ban installed)
   - `[ -f "$CONFIG_FILE" ]` (config.env exists)
   - `[ -f "$PROJECTS_FILE" ]` (projects.conf exists)
   - `[ -d "$CC_TMUX_DIR/lib" ]` (runtime lib deployed)
   Prints check/cross for each item. Does NOT verify SSH is running (that's Phase 2 territory for hardened setup).

Create `install.sh` as the single entry point for the installer. Structure:

**Header section (lines 1-20):**
- Shebang: `#!/usr/bin/env bash`
- CRLF self-healing block at the very top (before set -euo pipefail):
  ```bash
  # Self-heal CRLF if cloned on Windows without .gitattributes
  if head -1 "$0" | grep -q $'\r'; then
      echo "Fixing Windows line endings..."
      find "$(dirname "$0")" -name "*.sh" -exec sed -i 's/\r$//' {} +
      exec bash "$0" "$@"
  fi
  ```
- Then: `set -euo pipefail`

**Step functions defined in install.sh (not in lib/ -- these are installer-specific):**

1. `step_preflight()` -- Step 1. Calls `require_wsl`, `require_sudo`, `check_internet`. Checks bash version >= 5 with `BASH_VERSINFO[0]`. Logs ok for each check.

2. `step_detect_environment()` -- Step 4 (after deps). Calls `detect_windows_username`. If returns 1 and `NONINTERACTIVE` is false: prompts with `read -rp "  What is your Windows username? " WIN_USERNAME`. If `NONINTERACTIVE` is true, tries `/mnt/c/Users/` directory listing and picks the first non-system match, exits with error if none found. Calls `detect_wsl_distro` to get `WSL_DISTRO`. Calls `detect_win_home "$WIN_USERNAME"` to get `WIN_HOME`. Logs detected values with `log_ok`.

3. `step_configure()` -- Step 5. Calls `ensure_config_dir`. Calls `write_config` with all detected values:
   ```
   CC_TMUX_VERSION "2.0.0"
   CC_TMUX_INSTALL_DATE "$(date +%Y-%m-%d)"
   CC_TMUX_REPO "$SCRIPT_DIR"
   WIN_USERNAME "$WIN_USERNAME"
   WIN_HOME "$WIN_HOME"
   WSL_DISTRO "$WSL_DISTRO"
   TUNNEL_PROVIDER "ngrok"
   SESSION_NAME "work"
   ```
   Then calls `setup_ngrok_token` (from deps.sh).
   Then runs interactive project setup:
   - If `$PROJECTS_FILE` exists and is non-empty, logs ok with count and skips (idempotent).
   - Otherwise, if `NONINTERACTIVE` is true: adds one default project `home|$WIN_HOME/Documents` and returns.
   - Otherwise: interactive loop. Prints instructions. For each project:
     - `read -rp "  Project name (or 'done' to finish): " name`
     - Break on 'done' or empty
     - `read -rp "  Folder path [$WIN_HOME/Documents]: " path` with default
     - Validate path exists (warn if not, ask to add anyway)
     - `add_project "$name" "$path"`
   - If zero projects added, add default `home|$WIN_HOME/Documents`

4. `step_setup_system()` -- Step 6. Calls `setup_ssh_config`, `setup_sudoers`, `setup_bashrc_hook` (all from setup.sh).
