Create `startup.sh` in the repo root -- the v2 replacement for V1/startup.sh. This is the script users run to start their workspace.

```bash
#!/usr/bin/env bash
# ============================================
# startup.sh -- Start cc-tmux workspace
#
# Starts SSH, tunnel, and attaches to tmux.
# Deployed to ~/startup.sh by the installer.
#
# Usage: bash ~/startup.sh
# ============================================

set -euo pipefail

# Source runtime libraries from deployed location
CC_TMUX_DIR="$HOME/.cc-tmux"

if [[ ! -d "$CC_TMUX_DIR/lib" ]]; then
    echo "cc-tmux not installed. Run install.sh first."
    exit 1
fi

source "$CC_TMUX_DIR/lib/common.sh"
source "$CC_TMUX_DIR/lib/config.sh"
source "$CC_TMUX_DIR/lib/tunnel/provider.sh"
```

**Main sequence:**

1. **Print banner:**
   ```
   echo ""
   echo "========================================"
   echo "  CC-TMUX -- Starting workspace"
   echo "========================================"
   echo ""
   ```

2. **Start SSH server** (same as V1 but with logging):
   ```bash
   if sudo -n service ssh start 2>/dev/null; then
       log_ok "SSH server running"
   else
       log_warn "Could not start SSH (may already be running)"
   fi
   ```

3. **Load and start tunnel** (warn but continue on failure -- per locked decision):
   ```bash
   if load_tunnel_provider; then
       if tunnel_start; then
           # tunnel_start already logs success and prints URL
           :
       else
           log_warn "Tunnel failed to start -- workspace will run locally"
           log_hint "Check: ngrok config check"
           log_hint "Or run later: source ~/.cc-tmux/lib/tunnel/provider.sh && load_tunnel_provider && tunnel_start"
       fi
   else
       log_warn "Tunnel provider not available -- workspace will run locally"
   fi
   ```

4. **tmux session management:**
   Read session name from config with fallback: `SESSION_NAME=$(get_config "SESSION_NAME" 2>/dev/null) || SESSION_NAME="work"`
   ```bash
   if tmux has-session -t "$SESSION_NAME" 2>/dev/null; then
       log_ok "Attaching to existing session: $SESSION_NAME"
   else
       tmux new-session -d -s "$SESSION_NAME"
       log_ok "Created new session: $SESSION_NAME"
       # Phase 4 will add project tab creation here
   fi
   ```

5. **Display connection info before attaching** (per locked decision: "Display tunnel address on every tmux attach"):
   ```bash
   echo ""
   echo "========================================"
   if tunnel_url &>/dev/null; then
       local addr
       addr=$(tunnel_url)
       echo "  Workspace is running!"
       echo ""
       echo "  SSH from phone:"
       echo "    Host: ${addr%:*}"
       echo "    Port: ${addr##*:}"
       echo "    User: $USER"
       echo ""
       echo "  Or: ssh -p ${addr##*:} $USER@${addr%:*}"
   else
       echo "  Workspace is running (local only)"
       echo "  Tunnel not connected -- check: tunnel_status"
   fi
   echo "========================================"
   echo ""
   ```

   NOTE: Since this is at the top-level of the script (not inside a function), replace `local addr` with just `addr=...` (local is only valid inside functions). Alternatively, wrap the entire main sequence in a `main()` function and call it -- follow the pattern from install.sh which uses a `main()` function.

   DECISION: Wrap everything in a `main()` function (matches install.sh pattern). This way `local` variables work correctly.

6. **Attach to tmux** (exec replaces the shell process):
   ```bash
   exec tmux attach -t "$SESSION_NAME"
   ```

The file should be approximately 80-100 lines. Clean, well-commented, follows the established patterns.

IMPORTANT: The `tunnel_url` function is only available if `load_tunnel_provider` succeeded. Guard the connection info display: check if `tunnel_url` is a defined function before calling it (`declare -f tunnel_url &>/dev/null`).
