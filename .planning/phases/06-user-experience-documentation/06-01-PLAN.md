tunnel_url() {
    # Returns "host:port" string from tunnel.env
    # Returns 0 on success, 1 if no tunnel data
    if [[ -f "$TUNNEL_ENV" ]]; then
        source "$TUNNEL_ENV"
        if [[ -n "${TUNNEL_HOST:-}" ]]; then
            echo "${TUNNEL_HOST}:${TUNNEL_PORT}"
            return 0
        fi
    fi
    return 1
}

   - Arguments: `-d $wsl_distro -- bash -lc "~/startup.sh"` (use the config WSL_DISTRO value, not hard-coded "Ubuntu")
   - IconLocation: `C:\Windows\System32\wsl.exe,0`
5. CRITICAL: Escape ALL PowerShell dollar signs as `\$` in the double-quoted bash string (e.g., `\$WshShell`, `\$Shortcut`, `\$env:USERPROFILE`).
6. Check exit code: if 0, `log_ok "Desktop shortcut created: Claude Workspace"`. If non-zero, `log_warn "Could not create desktop shortcut"` + `log_hint "Create manually: see README.md"`.

Pattern reference: V1/setup-shortcut.ps1 for the PowerShell COM logic. Research notes for dollar-sign escaping pitfall.
