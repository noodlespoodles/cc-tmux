Split context menu into CRLF-safe architecture: thin PS1 launcher + WSL-side bash helper with deploy-time path substitution.

- Converts Windows path to WSL path via `wslpath`

## Key Decisions

1. **__WSL_HOME__ placeholder pattern**: The PS1 template uses a literal `__WSL_HOME__` string that gets replaced with the actual WSL home path (e.g., `/home/ben`) at install time via `sed -i`. This gives PowerShell an absolute path to pass to `wsl.exe`, completely avoiding `bash -c` and tilde expansion issues.

2. **Direct script execution via wsl.exe**: `wsl.exe -- /home/ben/.cc-tmux/templates/open-in-cctmux.sh C:\path` executes the script directly. The bash script lives on the WSL filesystem with LF line endings, so CRLF can never corrupt it.
