  <action>
Create `bin/cc-tmux` as a bash script with case-based subcommand routing. Per user decision, use case statement (not getopts).

Structure:
```
#!/usr/bin/env bash
set -euo pipefail
CC_TMUX_DIR="$HOME/.cc-tmux"
```

Source libraries: common.sh, config.sh. Only source workspace.sh and tunnel/provider.sh when needed (lazy loading in relevant case branches).

**show_usage():** Print usage with all commands: start, stop, project {add|remove|list}, tunnel.

**case "${1:-}" in:**

1. `start)` -- `exec bash "$HOME/startup.sh"` (delegates to startup.sh)

2. `stop)` -- Per Claude's discretion (research recommends adding it for symmetry with start):
   - Read session name from config with fallback to "work"
   - Source tunnel/provider.sh, call `load_tunnel_provider && tunnel_stop` (ignore errors)
   - `tmux kill-session -t "$session_name" 2>/dev/null` (ignore errors)
   - Log "Workspace stopped"

3. `project)` -- shift, then nested case:
   - `add)` -- shift, validate $1 (name) and $2 (path) exist, validate path is absolute (starts with /). Call `add_project "$name" "$path"`. Then if `tmux has-session -t "$session_name"` succeeds, create live window: `tmux new-window -t "$session_name" -n "$name" -c "$path"` and `tmux send-keys -t "$session_name:$name" "powershell.exe" Enter`, log_ok "Window '$name' created in active session"
   - `remove)` -- shift, validate $1 (name) exists. Call `remove_project "$name"`. Then if session active AND window exists (`tmux list-windows -t "$session_name" -F '#{window_name}' | grep -qx "$name"`), kill it: `tmux kill-window -t "$session_name:$name"`, log_ok "Window '$name' removed from active session"
   - `list)` -- call `list_projects`
   - `*)` -- print "Usage: cc-tmux project {add|remove|list}", exit 1

4. `tunnel)` -- source tunnel/provider.sh, call `load_tunnel_provider && tunnel_status`

5. `help|--help|-h)` -- call show_usage

6. `*)` -- call show_usage, exit 1

Session name: read via `get_config "SESSION_NAME" 2>/dev/null || echo "work"` at the point where it's needed (project add/remove/stop), not at top level.

Source workspace.sh in the `project` case branch (for potential future use and to keep the library loaded consistently when doing project operations).
