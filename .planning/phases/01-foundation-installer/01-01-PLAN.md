Create `lib/common.sh` with `set -euo pipefail` awareness (this file is sourced, not executed directly, so do NOT put set -euo pipefail at the top -- that belongs in the entry script that sources this).

The file must provide these exact functions:

1. `setup_colors()` -- Sets ANSI color variables (RED, GREEN, YELLOW, BLUE, BOLD, RESET). If `NO_COLOR` env var is set or stdout is not a TTY (`! -t 1`), set all to empty strings. Use escape codes: RED=`\033[0;31m`, GREEN=`\033[0;32m`, YELLOW=`\033[0;33m`, BLUE=`\033[0;34m`, BOLD=`\033[1m`, RESET=`\033[0m`.

2. `log_ok()` -- Prints `  [ok] $*` with green `[ok]`
3. `log_error()` -- Prints `  [error] $*` with red `[error]`
4. `log_warn()` -- Prints `  [warn] $*` with yellow `[warn]`
5. `log_hint()` -- Prints `       $*` with blue text (indented to align under message text)
6. `log_step()` -- Takes two args: step_number and message. Prints `[N/TOTAL_STEPS] message` in bold. Uses a global `TOTAL_STEPS` variable (set by the caller).

7. `install_package()` -- Takes package name. Checks `dpkg -l "$pkg" 2>/dev/null | grep -q "^ii"`. If already installed, `log_ok "$pkg already installed"`. Otherwise runs `sudo apt install -y "$pkg"` with error handling that calls `log_error` and `log_hint "Try: sudo apt update && sudo apt install -y $pkg"` on failure.

8. `add_bashrc_block()` -- Takes a block name and content. Uses sentinel markers `# CC-TMUX:$name:START` and `# CC-TMUX:$name:END`. Checks with `grep -qF "CC-TMUX:$name:START"`. If found, replaces the block between sentinels using `sed`. If not found, appends the block with sentinels. This makes it idempotent for both first run and re-runs.

9. `deploy_file()` -- Takes src and dst. Creates parent directory with `mkdir -p`. Copies file with `cp`. Sets permissions to 644 unless a third arg specifies otherwise.

10. `require_wsl()` -- Checks `/proc/version` for "microsoft" (case-insensitive). Exits with clear error if not in WSL2.

11. `require_sudo()` -- Tests `sudo -v 2>/dev/null`. Exits with clear error if no sudo access.

12. `check_internet()` -- Pings `archive.ubuntu.com` with `ping -c 1 -W 3`. Returns 0 on success, logs warning (not error) on failure.

Call `setup_colors` at the end of the file so colors are initialized when the file is sourced.

Create `lib/detect.sh` with these functions:

1. `detect_windows_username()` -- Tiered fallback detection:
   - Method 1: `timeout 5 cmd.exe /C "echo %USERNAME%" 2>/dev/null | tr -d '\r\n'`. Validate result is non-empty and is not literally `%USERNAME%`.
   - Method 2: Parse `/mnt/c/Users/` directory. List all directories, filter out system dirs (Default, "Default User", Public, "All Users", desktop.ini). If exactly one candidate, return it. If multiple candidates, return failure (caller will prompt).
   - Method 3: Return 1 (failure). The caller (install.sh) handles the interactive fallback prompt.
   - On success: echo the username and return 0. On failure: return 1 with no output.

2. `detect_wsl_distro()` -- Reads `WSL_DISTRO_NAME` env var if set. Falls back to parsing `/etc/os-release` for `NAME=`. Echoes the distro name (e.g., "Ubuntu"). Returns 1 if detection fails.

3. `detect_win_home()` -- Takes a Windows username as arg. Constructs path `/mnt/c/Users/$1`. Validates directory exists. Echoes the path. Returns 1 if directory not found.

Create `lib/config.sh` with these functions:

1. `CC_TMUX_DIR` variable set to `$HOME/.cc-tmux`
2. `CONFIG_FILE` variable set to `$CC_TMUX_DIR/config.env`
3. `PROJECTS_FILE` variable set to `$CC_TMUX_DIR/projects.conf`

4. `ensure_config_dir()` -- Creates `$CC_TMUX_DIR` with `mkdir -p`. Sets permissions 700.

5. `write_config()` -- Takes key-value pairs as arguments (e.g., `write_config WIN_USERNAME "Ben" WIN_HOME "/mnt/c/Users/Ben"`). If config.env exists, updates existing keys and appends new ones. If not exists, creates with header comment `# Generated by cc-tmux installer -- hand-editable`. Each pair written as `KEY="value"` (quoted values for safety with spaces). Adds `# Last updated: $(date +%Y-%m-%d)` to the header.

6. `read_config()` -- Sources `$CONFIG_FILE` if it exists. Returns 1 with `log_error` if file missing.

7. `get_config()` -- Takes a key name, echoes its value from the sourced config. Returns 1 if key not set.

8. `add_project()` -- Takes name and path. Validates name is non-empty and contains no pipe character. Validates path format (must start with `/`). Appends `name|path` to `$PROJECTS_FILE`. If a project with that name already exists, logs warning and skips (idempotent).

9. `remove_project()` -- Takes name. Removes the line starting with `name|` from `$PROJECTS_FILE` using grep -v. Logs ok or warning if not found.

10. `list_projects()` -- Reads `$PROJECTS_FILE` line by line using `while IFS='|' read -r name path`. Prints formatted list. Returns 1 if file is empty or missing.

11. `project_count()` -- Echoes the number of lines in `$PROJECTS_FILE` (0 if file missing).

Both files assume `lib/common.sh` has already been sourced by the caller (they use log_ok, log_error, etc. without sourcing common.sh themselves).
